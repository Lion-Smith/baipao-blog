# 前端个人博客从零上线全流程：GitHub Pages + 阿里云域名

## 一、成果预览

最终通过以下地址访问你的个人网站：

- **GitHub Pages 默认域名**：`https://lion-smith.github.io/baipao-blog/`
- **自定义域名**：`https://www.baipao.cyou`

---

## 二、本地代码提交到 GitHub

### 2.1 初始化本地仓库

```bash
cd 你的项目目录
git init
git add .
git commit -m "初始化项目"
```

### 2.2 关联远程仓库

在 GitHub 上先创建一个空仓库（不要勾选 README / .gitignore），然后：

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

> **注意**：GitHub 默认分支名为 `main`，本地旧版 Git 可能为 `master`，建议统一为 `main`。

### 2.3 添加 .gitignore

```bash
# 示例：排除 IDE 配置和本地缓存
.claude/
.vscode/
node_modules/
```

## 三、启用 GitHub Pages

### 3.1 方式一：从 main 分支部署（推荐）

1. 打开仓库 **Settings → Pages**
2. **Source** 选 `Deploy from a branch`
3. **Branch** 选 `main`，目录选 `/ (root)`
4. 点击 **Save**

3~5 分钟后访问 `https://你的用户名.github.io/仓库名/` 即可看到网站。

### 3.2 方式二：从 gh-pages 分支部署

```bash
git checkout -b gh-pages
git push -u origin gh-pages
```

传统方式，部分老项目仍在使用。效果相同。

### 3.3 为什么选择 main 分支根目录？

- 单页项目（只有一个 `index.html`）维护简单
- 不需要切分支，推送即部署
- GitHub Actions 支持更好

## 四、购买阿里云域名

### 4.1 选域名

1. 打开 [阿里云域名注册](https://wanwang.aliyun.com/domain/)
2. 搜索你想要的域名（本文以 `baipao.cyou` 为例）
3. 加入购物车 → 填写所有人信息 → 支付

### 4.2 实名认证

**国内域名强制要求实名，否则无法解析！**

1. 进入 [阿里云域名控制台](https://dc.console.aliyun.com/)
2. 找到你的域名 → 点击 **实名认证**
3. 上传身份证正反面
4. 等待审核（通常 **1~3 个工作日**，实际一般几小时）

> `.cyou` 后缀属于国际域名，实名审核较快。`.cn` 后缀审核最严。

### 4.3 备案？不需要

GitHub Pages 服务器在美国，不属于中国境内服务器，**不受 ICP 备案制度约束**。域名实名后直接解析即可。

## 五、域名 DNS 解析绑定

### 5.1 阿里云添加 DNS 记录

进入 [阿里云 DNS 控制台](https://dcdns.console.aliyun.com/) → 找到域名 → **解析设置** → **添加记录**：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| `CNAME` | `www` | `lion-smith.github.io` |
| `A` | `@` | `185.199.108.153` |
| `A` | `@` | `185.199.109.153` |
| `A` | `@` | `185.199.110.153` |
| `A` | `@` | `185.199.111.153` |

**记录说明**：

| 记录 | 作用 |
|------|------|
| `www` CNAME | 访问 `www.你的域名` 时跳转到 GitHub Pages |
| `@` A 记录 × 4 | 访问 `你的域名`（不带 www）时也能正常访问 |

### 5.2 GitHub Pages 配置自定义域名

在仓库根目录创建 `CNAME` 文件：

```bash
echo "www.baipao.cyou" > CNAME
git add CNAME
git commit -m "添加自定义域名"
git push
```

然后去仓库 **Settings → Pages → Custom domain** 填入 `www.baipao.cyou` → Save。

### 5.3 验证 DNS 是否生效

```bash
# Windows / Mac / Linux 通用
nslookup www.baipao.cyou
```

如果返回 GitHub 的 IP 地址（`185.199.x.x`），说明解析已生效。如果没有，等 **5~30 分钟** 再试。

### 5.4 启用 HTTPS

DNS 生效后，回到 GitHub Pages 设置，勾选 **Enforce HTTPS**。GitHub 会自动通过 Let's Encrypt 签发免费 SSL 证书，通常几分钟到 1 小时内完成。

## 六、完整操作时间线

| 步骤 | 耗时 |
|------|------|
| 本地代码推 GitHub | 1 分钟 |
| 启用 Pages 部署 | 3~5 分钟 |
| 阿里云购买域名 | 5 分钟 |
| 域名实名认证 | 1 小时 ~ 3 天 |
| 添加 DNS 记录 | 2 分钟 |
| DNS 解析生效 | 5~30 分钟 |

## 七、常见问题排查

### Q1：GitHub Pages 显示 `Domain's DNS record could not be retrieved`

**原因 1**：Custom domain 填了 `baipao.cyou`，但 `CNAME` 文件写的是 `www.baipao.cyou`——两者必须完全一致。

**原因 2**：DNS 记录还没配或还没生效，先用 `nslookup` 验证。

**原因 3**：域名还没完成实名认证。

### Q2：勾选 Enforce HTTPS 后报错

刚填好域名后**先不要勾选**，等 GitHub 自动签发完证书（DNS check 变绿）后再勾。太早勾选会导致检查失败。

### Q3：Git push 报 `Updates were rejected`

远程仓库有本地没有的提交（比如在 GitHub 网页端直接修改了文件）。

```bash
git pull --rebase
git push
```

### Q4：移动端访问布局错乱

检查 `index.html` 是否包含 viewport meta 标签：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### Q5：百度地图无法显示

1. 检查 API Key（AK）是否正确
2. 确认在百度地图开放平台创建应用时选了 **浏览器端** 类型
3. 检查 Referer 白名单是否包含 `*.baipao.cyou` 和 `*.github.io`

### Q6：网站打开空白或 404

- 仓库根目录必须有 `index.html`（不是放在子文件夹里）
- Pages 设置的部署分支和目录是否正确
- 仓库必须是 Public（私有仓库需要 GitHub Pro）
